# RoleCard (RC) 语言

**版本**：v1.0 正式版（已冻结）
**状态**：由 [77](https://github.com/had77guai) 设计并维护

---

## 这是什么？

RoleCard (RC) 语言是一种专为 **AI 角色扮演** 设计的**结构化提示词语言**。它用人类一眼就能看懂的纯文本，让你像填一张高级人物设定集一样，编写出能被 AI 稳定解析的复杂角色定义。

- **零门槛上手**：只要你会用 `#` 写标题，用 `-` 列清单，就能在五分钟内写出第一份角色文件。
- **极高的上限**：从轻量级状态机、规则优先级标签，到模板继承、可复用行为模块、输出格式锚定，RC 语言原生支持现代 AI 角色扮演所需的高级特性。

---

## 为什么选择 RC？

| 痛点 | RC 的解决方式 |
|------|--------------|
| YAML/JSON 符号多、易出错 | 仅用 `#` `-` `:` `//` `「」` `[]` 六种符号 |
| 长篇散文与结构化数据难共存 | 扁平化章节制，背景故事和属性表天然分离 |
| 状态变化难以定义 | 内置轻量级状态机，纯自然语言描述转换条件 |
| 规则冲突时 AI 行为难以预测 | `[必须]` / `[重要]` / `[建议]` 三层优先级标签 |
| 角色无法复用已有设定 | 支持模板继承和可插拔行为模块 |

## 30秒预览

```rc
# 基础信息
姓名: 艾莉西亚
年龄: 17
身份: 没落贵族家系的独女

# 性格
核心特质: 外冷内热，表面高傲实则渴望被理解

# 行为准则
- [必须] 绝不使用粗俗语言
- [重要] 保持优雅坐姿，避免夸张动作
- [建议] 偶尔引用古典文学

# 状态管理
初始状态: 日常公开

状态 日常公开:
  默认行为: 保持礼貌得体的社交形象
  转换规则:
    - 当 用户是亲密好友且周围没有其他人时 -> 私下亲密
```

这份文件同时对人友好（你能直接读懂这个角色的性格和状态切换），也对机器友好（解析器能无歧义地编译成可执行的提示词）。

## 进阶能力

当你的角色需要**更复杂的动态行为**时，RC 语言可以和 YAML 动态引擎混合使用：

- **RC 做静态角色定义**：性格、背景、行为准则、状态切换逻辑。
- **YAML 做动态数值计算**：实时情感数值、事件-数值映射、精密生成流水线。

详见 [RC+YAML 融合架构说明](./RC-YAML融合架构说明.md)。
